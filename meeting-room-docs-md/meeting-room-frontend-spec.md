# 会议室预定系统前端说明文档

## 1. 文档目的
本文档用于说明会议室预定系统前端侧的业务目标、功能范围、页面结构、交互逻辑、技术选型建议与非功能要求，供产品、设计、前端开发、测试和项目管理人员统一理解。

## 2. 项目背景
企业内部通常存在会议室资源分散、占用情况不透明、临时占用冲突、预定记录难以追踪等问题。会议室预定系统的前端目标是为用户提供一个**简单、直观、可视化**的操作入口，使其可以快速完成：

- 查询空闲会议室
- 按时间、容量、设备筛选会议室
- 发起预定、取消、修改预定
- 查看个人预定记录
- 管理员查看会议室状态与异常占用情况

## 3. 用户角色
### 3.1 普通用户
- 查询会议室空闲状态
- 创建会议室预定
- 查看、修改、取消自己的预定
- 进行签到或释放未使用会议室（可选）

### 3.2 管理员
- 管理会议室基础信息
- 管理会议室设备标签
- 查看预定记录与使用统计
- 处理冲突、封禁时段、维护状态

### 3.3 前台/运营人员（可选）
- 协助线下会议安排
- 查看当天会议室排期
- 处理临时占用和加时申请

## 4. 核心业务目标
### 4.1 提高预定效率
用户在 1~3 分钟内即可完成一次会议室查询与预定。

### 4.2 降低冲突率
通过实时状态展示、冲突校验、预定规则提示，减少重复预定和资源浪费。

### 4.3 提升资源利用率
通过可视化日历、筛选器、签到与释放机制，提升会议室使用透明度。

### 4.4 支持扩展
为后续接入企业通讯录、消息通知、门禁/签到设备、数据看板等能力预留前端扩展点。

## 5. 前端功能范围
## 5.1 登录与身份识别
- 支持账号登录 / 单点登录（SSO）
- 登录后根据角色展示不同菜单
- 支持记住登录态、Token 续期、权限失效跳转

## 5.2 首页 / 工作台
- 展示“快速预定”入口
- 展示“今日我的会议”
- 展示“当前空闲会议室推荐”
- 展示会议室使用公告、维护提醒

## 5.3 会议室列表页
- 以列表或卡片形式展示会议室
- 显示基础信息：名称、楼层、容量、设备、当前状态
- 支持多条件筛选：
  - 日期
  - 起止时间
  - 所在楼宇/楼层
  - 容量
  - 设备（投影、视频会议、白板等）
  - 状态（空闲/占用/维护）
- 支持排序：
  - 按容量
  - 按距离
  - 按匹配度
  - 按最早可用时间

## 5.4 会议室详情页
- 展示会议室图片、位置、设备、容量、开放时间
- 展示时间轴 / 日历视图
- 展示该会议室当天或指定日期的预定情况
- 支持直接从详情页发起预定

## 5.5 预定创建页 / 弹窗
- 输入会议主题
- 选择日期、开始时间、结束时间
- 选择参会人数
- 选择会议室
- 选择设备需求（可选）
- 备注信息（可选）
- 发起预定前展示冲突检查结果
- 提交成功后展示预定编号与状态

## 5.6 我的预定
- 查看我的全部预定
- 按状态筛选：待开始、进行中、已结束、已取消
- 支持修改时间、取消预定、再次预定
- 支持查看签到状态和提醒状态

## 5.7 日历视图
- 支持天 / 周维度查看会议室占用情况
- 支持按楼层、会议室维度切换
- 支持拖拽查看时间占用区间（高级版）
- 支持点击空白时间段快速发起预定

## 5.8 管理后台
- 会议室管理：新增、编辑、停用、维护
- 设备标签管理
- 预定规则管理：提前预定时长、最短/最长预定时长、取消规则
- 统计报表查看：利用率、取消率、空置率、峰值时段

## 7. 核心交互流程
### 7.1 用户查询并预定会议室
1. 用户进入首页或会议室列表页  
2. 选择日期、时间、人数、设备条件  
3. 系统返回符合条件的空闲会议室  
4. 用户查看详情后选择会议室  
5. 填写会议主题并提交  
6. 系统校验冲突并创建预定  
7. 前端提示成功，跳转到“我的预定”详情页  

### 7.2 用户取消预定
1. 用户进入“我的预定”  
2. 找到目标预定  
3. 点击取消  
4. 系统校验是否在可取消时间范围内  
5. 取消成功后前端更新状态并释放时间段  

### 7.3 管理员维护会议室
1. 管理员进入会议室管理页面  
2. 编辑会议室信息或标记维护时段  
3. 系统更新会议室可用状态  
4. 前端相关列表与详情页同步刷新  

## 8. 前端界面设计建议
## 8.1 设计原则
- 清晰：核心动作应一眼可见
- 快速：常用操作不超过 3 次点击
- 一致：列表、详情、弹窗、状态标识统一
- 可视化：尽量使用颜色、标签、时间轴表达状态
- 易扩展：支持后续增加签到、审批、消息提醒

## 8.2 状态颜色建议
- 空闲：绿色
- 已预定：蓝色
- 使用中：橙色
- 维护中：灰色
- 冲突 / 不可用：红色

## 8.3 关键组件建议
- 顶部筛选栏
- 时间选择器
- 会议室卡片
- 时间轴组件
- 日历组件
- 状态标签 Tag
- 表单弹窗 / 抽屉
- Toast 提示
- 空状态页与异常页

## 9. 推荐技术方案
## 9.1 技术栈建议
- 前端框架：Vue 3 / React
- 构建工具：Vite
- UI 组件库：Element Plus / Ant Design / Arco Design
- 状态管理：Pinia / Redux Toolkit
- 路由：Vue Router / React Router
- 请求库：Axios
- 时间处理：dayjs
- 图表统计：ECharts
- 国际化：vue-i18n / react-intl（如有多语言需求）

## 9.3 前后端交互建议
- 统一 RESTful API 风格
- 统一响应结构：code / message / data
- 对时间字段统一使用 ISO 8601 或时间戳
- 对会议室状态、预定状态使用枚举
- 对冲突类错误提供明确错误码，便于前端提示

## 11. 前端权限设计
### 11.1 菜单权限
根据角色控制是否显示管理菜单。

### 11.2 页面权限
无管理员权限的用户不能访问管理端页面。

### 11.3 按钮权限
如新增会议室、删除会议室、修改规则等按钮只对管理员显示。

### 11.4 数据权限
普通用户只能查看自己的预定详情；管理员可查看全量数据。

## 12. 异常与边界场景
- 用户选择的时间已被他人抢先预定
- 开始时间晚于结束时间
- 参会人数超过会议室容量
- 会议室维护中不可预定
- 用户网络异常导致提交失败
- Token 过期需要重新登录
- 提交成功但消息通知发送失败（前端需提示“预定成功，通知稍后补发”）

## 13. 非功能要求
### 13.1 性能
- 列表页首屏加载建议 < 2 秒
- 筛选结果响应建议 < 1 秒（在接口正常情况下）
- 支持分页和懒加载

### 13.2 易用性
- 支持键盘输入与快捷操作
- 表单校验提示清晰
- 支持空状态说明和引导

### 13.3 兼容性
- 支持 Chrome、Edge 主流浏览器
- 管理端可优先适配桌面端
- 用户端可选适配移动端 H5

### 13.4 可维护性
- 组件职责清晰
- 公共业务组件可复用
- 类型定义统一
- 接口封装统一

## 16. 参考案例总结
可参考的成熟产品实践包括：
- Google Workspace 的会议室资源预定与房间推荐机制：强调楼宇、楼层、容量和设备属性，并支持基于地点与设备的智能推荐。
- Microsoft Teams Panels 的门口屏预定模式：强调房间状态一眼可见、现场快速预定、二维码辅助预定。
- Robin 的会议空间管理模式：强调按容量/设备匹配、占用分析、空间利用率统计。

这些实践说明一个成功的会议室预定前端通常需要同时具备：
1. 快速筛选能力  
2. 明确的空闲状态可视化  
3. 简洁的预定流程  
4. 良好的移动端或现场端扩展能力  
5. 可追踪的数据统计入口
